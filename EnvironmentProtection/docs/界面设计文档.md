# 绿法通 - 界面设计文档

## 一、设计原则

### 1.1 设计理念
- **简洁高效**：界面简洁，信息层次分明，操作路径短
- **统一风格**：小程序与Web端保持视觉风格统一
- **角色适配**：根据用户角色动态展示功能和内容
- **移动优先**：小程序端采用单手操作友好的设计

### 1.2 设计规范

#### 色彩系统
| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #28A745 | 环保绿，代表环保主题 |
| 辅助色 | #20C997 | 清新绿，用于渐变和强调 |
| 警示色 | #FFC107 | 黄色，用于提醒 |
| 危险色 | #DC3545 | 红色，用于错误和紧急 |
| 背景色 | #F0F7F2 | 浅绿底色，清新舒适 |
| 文字主色 | #333333 | 主要文字 |
| 文字辅色 | #666666 | 次要文字 |
| 文字弱色 | #999999 | 占位文字 |

#### 字体规范
- 标题：16-20px，加粗
- 正文：14px，常规
- 辅助：12px，常规
- 行高：1.5倍

## 六、交互规范

### 6.1 通用交互

| 场景 | 交互效果 |
|------|----------|
| 按钮点击 | 500ms内响应，显示loading状态 |
| 表单提交 | 防重复提交，禁用按钮并显示加载 |
| 列表加载 | 分页加载，每页20条，支持滚动加载 |
| 页面跳转 | 300ms内完成，必要时显示骨架屏 |
| 错误提示 | Toast提示，3秒后自动消失 |
| 成功提示 | 绿色Toast，可带图标 |

### 6.2 空状态

| 场景 | 空状态提示 |
|------|-----------|
| 无数据 | 插画 + "暂无数据" + 操作引导 |
| 网络错误 | 插画 + "网络异常" + 重试按钮 |
| 权限不足 | 插画 + "暂无权限" + 联系管理员 |

### 6.3 加载状态

| 场景 | 加载效果 |
|------|----------|
| 页面加载 | 骨架屏/全屏loading |
| 数据提交 | 按钮loading + 禁止操作 |
| 图片加载 | 占位图 + 渐进加载 |
